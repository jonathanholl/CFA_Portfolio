⚠ Case Synopsis — Operation Timber Trail

HSI (Homeland Security Investigations) agents executing a federal search warrant at a residential address seize multiple digital devices suspected of containing evidence related to fraud and trafficking of stolen goods. Among the items seized is a Dell Inspiron laptop belonging to the primary subject.

During a preliminary site walkthrough, agents photograph the interior of the residence. The living room contains a distinctive sectional sofa — later identified as a key evidentiary item potentially linking the subject to the sale of stolen property from a local furniture business.

The laptop is transported to the HSI digital forensics lab. Your role: Computer Forensic Analyst. You have been assigned to conduct a complete forensic examination of the seized hard drive and document all findings.

02
Forensic Procedure
Evidence Acquisition
🔒
Critical: Every step must be documented before proceeding. Improper handling voids the evidentiary value of the entire drive. Write-block BEFORE powering the device.
1

Physical Seizure & Packaging

At the scene, the laptop is powered off (hard shutdown — pull battery if possible to avoid write operations). Document the device with photographs showing all ports, serial numbers, and physical condition. Place in anti-static evidence bag. Apply tamper-evident evidence tape. Assign Evidence Item #E-001.

2

Chain of Custody Form (CF-1)

Complete the CF-1 form immediately at seizure. Document: date/time of seizure, seizing agent, case number, description of item, condition, and location seized from. Every subsequent transfer of the evidence must be logged — name, date, reason, and signature. This form is your evidentiary lifeline.

3

Lab Intake & Write Blocker

In the forensics lab, remove the hard drive from the laptop (document each screw removed if relevant). Attach a Tableau TX1 or Wiebetech write blocker via SATA/USB interface BEFORE connecting to your forensic workstation. Verify the write blocker LED indicates blocked write status. This physically prevents any data from being written to the evidence drive.

4

Forensic Image Acquisition (FTK Imager)

Launch FTK Imager → File → Create Disk Image → Physical Drive. Select the evidence drive. Output format: E01 (EnCase) or DD/RAW. Set destination to your forensic workstation (separate evidence-grade drive). Acquire the complete physical image including unallocated space and slack space.

5

Hash Verification

During acquisition, FTK Imager automatically generates MD5 and SHA-256 hashes of the source drive. Upon completion, verify the hash of the acquired image matches the source exactly. Any mismatch indicates corruption or tampering — the image is inadmissible. Document both hash values in your evidence log.

Hash Verification Example
# Source Drive Hash (computed at seizure)
MD5  : a3f8c2e91b4d7056fe2a1c3d8b9e0f14
SHA256: 4a7b1c9e2d5f0389ab4c6d1e8f2a3b7c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4

# Forensic Image Hash (computed after acquisition)
MD5  : a3f8c2e91b4d7056fe2a1c3d8b9e0f14  ✓ MATCH
SHA256: 4a7b1c9e2d5f0389ab4c6d1e8f2a3b7c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4  ✓ MATCH

# FTK Imager CLI verification
ftkimager \\.\PhysicalDrive1 E:\Evidence\TT-2024-047\E-001 --e01 --verify

Autopsy Analysis Workflow
1

Create New Case in Autopsy

New Case → Case Name: TT-2024-047 → Add Data Source → Disk Image (E01 file) → Run ingest modules: Hash Lookup, File Type ID, EXIF Parser, Keyword Search, Recent Activity.

2

File System Analysis

Examine NTFS partition structure. Browse C:\Users\[Subject]\Pictures, Downloads, Documents, AppData. Review the $MFT (Master File Table) for complete file history including deleted entries. Use Sleuth Kit's fls command to list all files including deleted ones marked with an asterisk (*).

3

Timeline Analysis

Use Autopsy's Timeline view to generate an activity timeline. Sort by file creation, modification, and access times. Look for clusters of activity — a large batch of image files accessed on the same day may indicate a specific event. Export the timeline to a CSV for documentation.

4

Keyword Searching

Build keyword lists in Autopsy: furniture store names, model numbers, "couch," "sectional," "receipt," "purchase." Also search for email addresses, phone numbers, and the specific store name identified in OSINT. Add regex patterns for common receipt formats.

5

Bulk Extractor Analysis

Run Bulk Extractor against the image for automatic extraction: bulk_extractor -o output_dir image.E01. Review email.txt, url.txt, and any detected credit card numbers that may corroborate a purchase from the furniture store.

04
Key Evidence
Deleted File Recovery

During the MFT analysis, Autopsy flags two deleted JPEG files in the user's Pictures directory. One of them — IMG_20240318_142237.jpg — is identified as an image of the subject's living room showing a distinctive gray sectional sofa with a visible manufacturer tag.

Recovery Method 1 — MFT Carving

The NTFS Master File Table retains metadata entries for deleted files. When a file is deleted, Windows marks the MFT record as available but does not immediately overwrite it. Autopsy identifies the deleted record, retrieves the cluster chain, and reassembles the file. Best case: file data still intact in original clusters.

Recovery Method 2 — File Carving

If the MFT entry is overwritten, use Foremost or PhotoRec to scan unallocated space for JPEG signatures. JPEG files begin with the magic bytes FF D8 FF and end with FF D9. Carving tools scan the entire drive for these byte sequences and reconstruct files regardless of filesystem metadata.

# Method 1 — Autopsy recovery via MFT
# In Autopsy: right-click deleted file → Extract File

# Method 2 — Foremost file carving
foremost -t jpg,png,pdf -i image.E01 -o ./carved_output/

# Method 3 — PhotoRec (interactive)
photorec image.E01

# Method 4 — Manual JPEG carve with dd
# Find JPEG header offset with grep/xxd, then:
dd if=image.E01 bs=512 skip=921440 count=512 of=recovered.jpg

# Extract EXIF from recovered image
exiftool recovered_couch.jpg

Lead Development
Investigative Pivot

The recovered photograph shows a gray sectional sofa in the subject's living room. A manufacturer label is partially visible. This becomes the investigative pivot point — tracing the couch to its purchase location.

STEP 01 — Visual Analysis
Identify couch brand from visible label
Zoom into the recovered high-resolution image. The manufacturer tag reads "Comfort Craft — Model: CC-4490-GRY." This is a distinctive mid-market sectional.
STEP 02 — Reverse Image Search
Google Images / TinEye search of the couch
Screenshot the couch from the recovered photo. Use reverse image search tools to find matching product listings. Identify the model as sold exclusively through small regional furniture retailers.
STEP 03 — Manufacturer Research
Contact or research Comfort Craft's dealer network
Comfort Craft's website lists authorized dealers. One dealer — "Rivera's Home Furnishings" — is located 4.2 miles from the GPS coordinates in the EXIF data.
STEP 04 — Cross-Reference Purchase Window
Correlate EXIF date with store records
EXIF timestamp: March 18, 2024. The photo was taken the same day as the likely delivery. Agents can now request purchase records from Rivera's Home Furnishings for that date range via subpoena.
STEP 05 — Browser Artifact Corroboration
Review browsing history from forensic image
Autopsy's web artifact module shows the subject visited riverafurniture.com on March 10, 2024 and viewed the exact CC-4490-GRY listing. Screenshots from the browser cache are extracted as supporting evidence.

Store Identification Methods

Business registry search: Search state business registration databases for furniture retailers near the GPS coordinates. Texas Secretary of State business search is publicly accessible.

Google Maps: Plot the EXIF GPS coordinates. Identify furniture stores within a 5-mile radius.

Yelp / Google Business: Search for "Rivera's Home Furnishings" to confirm address, phone, hours, and owner information.

Narrowing the Suspect Pool

Sales record subpoena: Investigators obtain a subpoena for Rivera's sales records for the CC-4490-GRY model in the March 2024 window. Only three units were sold.

Delivery records: The store's delivery log includes the delivery address. One matches the residence address associated with the subject.

Payment method: The transaction was paid via debit card linked to the subject's known bank account.

Advanced Additions
A

Create a Real Simulated Disk Image

Use a Linux system to create a 2GB virtual disk with planted evidence. Add a JPEG file, a PDF "receipt," and a text file, then "delete" them. Image the virtual disk with FTK Imager or dd, compute hashes, and run the full analysis against it. This gives you real, reproducible tool output — not screenshots from a tutorial.

B

Timeline Visualization

Export the Autopsy timeline CSV and build an interactive timeline using Python (plotly/matplotlib) or a free tool like TimelineJS. A visual evidence timeline dramatically improves the final report's readability and is impressive in a portfolio.

C

File Carving Demonstration

Plant a JPEG on your virtual disk, delete it, wipe the filename from the MFT, then use Foremost to carve it back. Document every step with screenshots. This directly demonstrates one of the most important CFA skills.

D

NIST Test Images

Use the NIST CFReDS (Computer Forensics Reference Data Sets) images available at cfreds.nist.gov. These are real forensic challenge datasets used for training and tool validation — perfect for a portfolio showing you can handle industry-standard data.

E

Maltego Link Analysis Chart

Build a Maltego entity link chart showing the relationships: subject → device → deleted file → EXIF GPS → furniture store → sales record → subject. Export as a PNG for the final report. This visual storytelling is extremely effective for non-technical audiences (like prosecutors or juries).

# Create simulated evidence disk on Linux
dd if=/dev/zero bs=1M count=2048 of=evidence_disk.img
mkfs.ntfs evidence_disk.img

# Plant evidence files
cp couch_photo.jpg /mnt/evidence/Users/Subject/Pictures/
cp furniture_receipt.pdf /mnt/evidence/Users/Subject/Documents/

# "Delete" evidence
rm /mnt/evidence/Users/Subject/Pictures/couch_photo.jpg
umount /mnt/evidence

# Hash and image the disk
sha256sum evidence_disk.img > evidence_disk.sha256
